How to Install Kali Linux in WSL (Windows 10 & Windows 11)

This guide explains how to install Kali Linux inside Windows using Windows Subsystem for Linux (WSL). This setup allows you to run Kali tools directly from Windows without a virtual machine.

Recommended: Use WSL 2 for better performance and full Linux kernel support.

System Requirements

  • Windows 10 version 2004+ or Windows 11
  • Virtualization enabled in BIOS
  • Administrator access

Step 1: Install WSL

Open PowerShell as Administrator and run:

wsl --install

Restart your system after installation completes.

To confirm WSL is installed:

wsl --status

Step 2: Install Kali Linux

Option 1: Install via Command Line (Recommended)

wsl --install -d kali-linux

This automatically downloads and installs Kali Linux.

Option 2: Install via Microsoft Store

  1. Open Microsoft Store
  2. Search for “Kali Linux”
  3. Click Install

Step 3: Launch Kali Linux

After installation: 
wsl -d kali-linux
Or search for “Kali Linux” in the Start menu. You will be prompted to create: - Username - Password This is your Linux user account.

Step 4: Update Kali Linux

After first login, always update: 
sudo apt update && sudo apt upgrade -y

Step 5: Verify WSL Version

wsl --list --verbose
You should see: Version: 2 If not, convert it: 
wsl --set-version kali-linux 2
Kali Linux is now successfully running inside WSL 2.

Common Issues & Fixes

Virtualization Not Enabled

Enable Intel VT-x or AMD-V from BIOS.

WSL Not Working

wsl --shutdown
Restart terminal.

EACCES Permission Errors with npm

Use NVM to manage Node inside Kali instead of system-wide installation.

Optional: Install Kali Tools Meta Package

Full Kali tools: 
sudo apt install -y kali-linux-large
Minimal tools: 
sudo apt install -y kali-linux-default

Why Use Kali in WSL?

  • No need for VirtualBox or VMware
  • Lightweight setup
  • Fast startup
  • Direct integration with Windows filesystem
  • Ideal for penetration testing labs

Install OpenCode CLI in Kali Linux (WSL)

After installing Kali Linux in WSL, you may want to install development tools like OpenCode CLI. Since OpenCode is a Node.js-based tool, we will use NVM (Node Version Manager) to avoid permission issues.

Step 1: Install NVM (Node Version Manager)

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.5/install.sh | bash
        source ~/.bashrc

Step 2: Install Latest LTS Version of Node.js

nvm install --lts
        nvm use --lts

Verify installation:

node -v
        npm -v

Step 3: Install OpenCode CLI Globally

npm install -g opencode-ai

No sudo is required when using NVM.

Step 4: Verify OpenCode Installation

opencode --version
OpenCode CLI is now successfully installed and ready to use inside Kali Linux (WSL).

Why Use NVM Instead of sudo npm?

  • Prevents EACCES permission errors
  • Avoids system-level conflicts
  • Allows multiple Node versions
  • Safer development environment

Conclusion

Installing Kali Linux in WSL is the most efficient way to run a cybersecurity-focused Linux distribution on Windows. With WSL 2, you get near-native performance without virtualization overhead. For best results:
  • Always use WSL 2
  • Keep Kali updated
  • Use NVM for Node-based tools
